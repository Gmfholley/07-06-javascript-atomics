What's the difference between alert() and console.log()?

Alert displays messages in a message box that halts the running of all other things until the user responds by hitting ok.

console.log works by placing messages in a browser console window, and the running of methods afterwards is not halted until the user responds.

What's one situation where you might use an `alert()` to tell the user something?

Alerts might be used to halt the execution of something or to require the user's attention before execution. For example, before deleting something or heading to a vulnerable website. Most of those scenarios sound like a confirm box rather than an alert box.

Just alerts might be useful to display messages that a user might otherwise miss before the page loads. For example, the goodbye message before clicking away from the site.

What's one way that `alert()` might get obnoxious?

No one likes to have to click things twice. Alerts would definitely be obnoxious if they were the de facto mode of communication with a user on any site.
